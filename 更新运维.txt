========================================
七、财务模式与 Yahoo Finance 代理
========================================
1) 数据源与接入方式
- 使用 Yahoo Finance Chart API：
  https://query1.finance.yahoo.com/v8/finance/chart/:symbol
- 无需 API Key
- 必须由服务端代理，不走浏览器直连
- 由 RAI 服务端提供统一接口：
  GET /api/quote/:symbol?range=...&interval=...

2) 默认参数与限制
- 默认 range：1mo
- 默认 interval：1d
- 允许的 range：
  1d, 5d, 1mo, 3mo, 6mo, 1y, 2y, 5y, 10y, ytd, max
- 允许的 interval：
  1m, 2m, 5m, 15m, 30m, 60m, 90m, 1d, 1wk, 1mo, 3mo

3) 符号兼容规则
- 已带后缀时直接使用：.SS / .SZ / .HK
- 自动补全：
  6xxxxx -> .SS
  0xxxxx / 3xxxxx -> .SZ
  4~5位纯数字 -> .HK

4) 缓存与时效
- 内存缓存 TTL：60 秒
- cache key：symbol + range + interval
- 相同请求 60 秒内优先命中缓存

5) 数据时效声明
- Yahoo Finance 免费数据中的 A 股 / 港股通常为延迟行情
- 不应视为逐笔实时成交数据
- 适合参考、分析、展示，不适合作为高频交易执行依据

6) 对 AI 的工具暴露
- 工具名：finance_quote
- 适用场景：
  证券代码、价格、涨跌幅、区间走势、历史 K 线、成交量
- 使用优先级：
  先 finance_quote
  再 web_search（新闻、公告、研报、背景信息）

7) 示例
- /api/quote/600519.SS?range=1y&interval=1d
- /api/quote/0700.HK?range=6mo&interval=1d

8) 排障思路
- 400：symbol / range / interval 非法
- 502：Yahoo 上游不可达、状态异常或返回数据无法解析
- 若浏览器直接调 Yahoo 报 CORS，这是预期；必须走 RAI 服务端代理

========================================
八、ZTX6D 登录与绑定流程（当前生效）
========================================
前置条件：
- 已配置 `ZTX6D_APP_ID`、`ZTX6D_APP_KEY`
- 建议配置 `PUBLIC_BASE_URL='https://rai.rick.quest'`
- 生产建议配置 `ZTX6D_CALLBACK_URL='https://rai.rick.quest/api/auth/ztx6d/callback'`

1) 状态检查（是否启用）
- 接口：`GET /api/auth/ztx6d/status`
- 返回 `enabled=true` 代表已可用；`enabled=false` 代表缺少 appid/appkey

2) 游客 / 新用户使用 ZTX6D 登录
- 前端按钮：登录页 `ZTX6D 登录`
- 后端入口：`GET /api/auth/ztx6d/start?return=/`
- 服务端流程：
  a. 调 ZTX6D `create_rt`（`once=1`）获取 `rt`
  b. 本地记录 `rt`（10 分钟有效）
  c. 302 跳转到 `https://passport.ztx6d.com/?rt=<rt>`
- 授权回调：`GET /api/auth/ztx6d/callback?rt=<rt>`
- 回调处理：
  a. 服务端调 `get_rt` 换取 `uid`
  b. 将 `ztx6d:<uid>` 映射/创建为本地 RAI 用户
  c. 签发 RAI JWT，并回跳 `return` 地址，附带 `rai_token`

3) 老用户绑定 ZTX6D（先登录 RAI）
- 前端入口：`设置 -> 账号绑定 -> 绑定 ZTX6D`
- 后端入口：`POST /api/auth/ztx6d/bind/start`（需携带当前登录 JWT）
- 返回：`redirectUrl`，前端跳转到 ZTX6D 授权页
- 回调后：把该 `ztx6d uid` 绑定到当前 RAI 用户；后续可直接用 ZTX6D 登录这个老账号

4) 常见回调错误
- `ztx6d_missing_rt`：回调里没有 rt
- `ztx6d_invalid_or_expired_rt`：rt 已过期/已消费/不存在
- `ztx6d_not_authorized`：用户尚未完成授权

5) 安全约束
- `ZTX6D_APP_KEY` 仅允许在服务端使用，禁止写入前端和 Git
- 同一个 `rt` 只可换取一次
